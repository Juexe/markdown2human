# Markdown2Human UI 设计简稿

## 1. 设计目标

- 让用户一眼理解“输入 Markdown，右侧得到可发送文本”的核心关系
- 让高频动作尽量少思考、少跳转、少学习成本
- 让界面观感轻盈、干净，适合长时间处理文本

## 2. 整体设计方向

界面采用桌面工具式单页布局，强调效率和可读性。视觉上以浅色、柔和层次和轻微玻璃感为主，避免复杂装饰，让注意力集中在输入内容、转换结果和少量必要设置上。

## 3. 页面结构

### 3.1 顶部控制区

- 放置所有与输出样式相关的轻量设置
- 设置项以横向排列为主，便于快速扫读和连续调整
- “关于”入口放在右侧次级位置，不干扰主流程

### 3.2 主工作区

- 采用左右双栏布局
- 左侧为 Markdown 输入区
- 右侧为转换结果区
- 两栏权重接近，形成“原文 / 结果”对照关系

### 3.3 弹层区

- “关于”信息使用居中弹层展示
- 弹层只承载补充信息，不放主流程操作

## 4. 信息层级

### 4.1 一级信息

- 输入内容
- 转换结果
- 复制结果

### 4.2 二级信息

- 打开文件
- 加载示例
- 清空内容
- 输出样式调节

### 4.3 三级信息

- 产品说明
- 版本与作者信息

## 5. 核心模块设计

### 5.1 输入区

- 作为主编辑区域，应具备明显的可输入感
- 标题明确为 “Markdown”
- 顶部操作按钮包括：打开、清空
- 文本区域应支持长文编辑，视觉上偏工具属性，强调内容密度

### 5.2 结果区

- 作为主输出区域，应与输入区形成一致结构
- 标题明确为 “转换结果”
- 复制按钮固定在区域头部，保持随手可用
- 文本区域应突出“已整理、更适合阅读”的感受，视觉上比输入区更柔和

### 5.3 设置区

- 使用“少量可控项”原则，不做复杂参数面板
- 下拉选项用于有限枚举配置，如列表符号、表格分隔方式、段落间距
- 开关项用于二元选择，如是否保留编号、链接、图片文本、代码块
- 每个设置项都应短标签、低干扰、可快速理解
