# Markdown2Human 产品需求文档

## 1. 产品概述

Markdown2Human 是一款面向日常沟通场景的文本整理工具。用户输入或导入 Markdown 内容后，产品将其转换为更适合在聊天软件、纯文本窗口和即时转发场景中阅读的文本结果。

产品核心价值是：不要求接收方理解 Markdown，也不要求用户手工清理格式，就能快速得到一份结构清楚、复制即用的文本内容。

## 2. 目标用户

- 需要把 Markdown 文档转发给同事、客户或群聊成员的知识工作者
- 习惯用 Markdown 记录内容，但发布渠道只适合纯文本的用户
- 需要快速整理说明文、会议纪要、项目简介等内容的个人用户

## 3. 核心使用场景

- 将 README、说明文档或操作指引整理后发到聊天工具
- 将会议纪要、周报或通知从 Markdown 转为更顺眼的纯文本
- 将带有简单表格、链接、图片说明和代码片段的内容快速压平后转发

## 4. 产品目标

- 让用户在几秒内完成 Markdown 到可读纯文本的转换
- 在尽量保留结构信息的前提下，降低 Markdown 语法对阅读的干扰
- 允许用户按使用习惯微调输出样式，而不是手工二次编辑
- 让结果可以一键复制，直接用于发送、保存或粘贴

## 5. 功能需求

### 5.1 内容输入

- 用户可直接在输入区粘贴或编辑 Markdown 内容
- 用户可打开本地 Markdown 文件并载入内容
- 用户可一键清空输入内容，开始新的转换

### 5.2 自动转换

- 当输入内容变化时，系统应自动刷新转换结果
- 输出结果应以纯文本形式展示，便于继续修改或直接复制
- 转换结果应尽量保留标题、列表、引用、表格、链接、图片说明和代码内容的阅读线索

### 5.3 输出样式调节

- 用户可切换无序列表符号
- 用户可控制表格内容的字段分隔方式
- 用户可控制段落间距的紧凑程度
- 用户可决定是否保留有序列表编号
- 用户可决定是否在链接文字后附带链接地址
- 用户可决定是否保留图片说明文字
- 用户可决定是否保留代码块内容

### 5.4 结果使用

- 用户可一键复制转换结果
- 复制成功后应给出明确反馈，避免重复操作

### 5.5 信息与状态

- 产品应提供简明的“关于”信息，帮助用户理解产品定位
- 产品应记住用户上次使用的转换偏好，减少重复设置
